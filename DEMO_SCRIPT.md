# 企业工单处理 Agent 演示脚本

## 演示目标

用 3-5 分钟展示这个项目不是普通聊天机器人，而是一个能处理业务流程的 Agent 应用。

## 启动项目

```bash
cd /Users/yaohe/Desktop/agent—study/ticket-agent-assistant
source ../.venv/bin/activate
streamlit run streamlit_app.py --server.port 8502
```

打开：

```text
http://localhost:8502
```

## 演示 1：知识类工单 + RAG

选择：

```text
T1001
```

讲解重点：

- 工单内容是“知识库没有找到产品介绍”。
- 系统会识别为 `knowledge_question`。
- Agent 会把投诉式表达改写成适合检索的问题。
- RAG 会检索企业知识库资料。
- DeepSeek 根据资料生成回复草稿。
- 人工审核后才发送。

可以强调：

```text
我没有让模型凭空回答，而是让知识类工单先检索企业知识库。
```

## 演示 2：高优先级系统问题 + 工具调用

选择：

```text
T1003
```

讲解重点：

- 系统会识别为 `system_issue`。
- 优先级为 `high`。
- 工具调用会建议负责团队。
- 回复草稿会体现优先处理。

可以强调：

```text
高优先级工单会触发工具建议，把工单转交给对应团队。
```

## 演示 3：自定义工单

勾选：

```text
输入自定义工单
```

输入：

```text
用户反馈：支付成功后没有开通课程权限。
```

讲解重点：

- 系统会识别为 `billing_question`。
- 优先级为 `high`。
- 工具调用建议商务/财务团队。
- 回复草稿不会乱承诺，只会说明已收到并将核实权限。

## 演示 4：日志与观测

切换到：

```text
日志与观测
```

讲解重点：

- 可以看到处理总数。

## 演示 5：评估报告

切换到：

```text
评估报告
```

讲解重点：

- 分类评估验证类别和优先级。
- 重复工单策略评估验证是否需要人工确认。
- 历史记忆检索评估验证是否找到了相似历史。
- 工具路由评估验证是否选择了正确工具。
- 一键评估总览显示整体是否通过。

## 1 分钟项目介绍

这个项目是一个企业工单处理 Agent。我用 LangGraph 把流程拆成读取工单、分类、重复检查、历史记忆、RAG 检索、工具调用、回复草稿生成、人工审核和日志保存等节点。

知识类工单会先 query rewrite，再检索企业知识库，避免模型凭空回答。高优先级或特殊类别工单会触发工具建议负责团队。回复发送前必须人工审核，处理结果会保存到 JSON 日志中，后续可以用于审计、记忆和评估。

项目还提供 Streamlit 页面和多类评估脚本，用来展示 Agent 的可交互性和可验证性。
