# 简历项目：企业工单处理 Agent

## 简历一句话

基于 LangGraph 和 Streamlit 构建企业工单处理 Agent，支持工单分类、知识库 RAG、历史记忆、工具调用、人工审核、日志持久化和多维评估。

## 简历项目描述

企业工单处理 Agent：

- 使用 LangGraph 设计多节点工单处理流程，包括读取工单、分类、重复工单确认、历史记忆检索、知识库 RAG、工具调用、回复生成、人工审核和日志保存。
- 对知识类工单进行 query rewrite，并复用企业知识库 RAG 检索资料，降低模型幻觉。
- 设计同一工单历史和相似历史工单记忆机制，使 Agent 能参考过去处理记录。
- 对重复工单和回复发送加入 human-in-the-loop，避免重复发送和误操作。
- 使用 Streamlit 构建可演示页面，支持自定义工单、草稿审核、日志搜索和评估报告展示。
- 编写分类、重复策略、历史记忆检索和工具路由评估脚本，输出 JSON 报告。

## 技术栈

```text
Python / Streamlit / LangGraph / LangChain / DeepSeek API / RAG / JSON / Tool Calling
```

## 可量化亮点

- 支持 12 个模拟工单场景。
- 覆盖知识咨询、系统问题、账号问题、费用问题、课程问题和用户投诉等类别。
- 具备 4 类评估：分类、重复策略、历史记忆检索、工具路由。
- 支持网页端人工审核和日志观测。

## 面试讲解重点

1. 为什么用 LangGraph 而不是普通函数串起来。
2. 为什么知识类工单要先 query rewrite 再 RAG。
3. 为什么重复工单确认要放在 RAG 和 LLM 调用之前。
4. 历史日志如何从“记录”变成“记忆”。
5. Tool Calling 如何帮助 Agent 选择负责团队或查询状态。
6. 为什么要做人工审核和评估脚本。
